# 模块目录规范

从本次开始，所有新功能按“独立模块目录”组织。

## 前端
- 路径：`frontend/src/modules/<module_name>/`
- 页面建议：`frontend/src/modules/<module_name>/pages/`
- API 建议：`frontend/src/modules/<module_name>/api/`

示例：
- `frontend/src/modules/script_optimizer/pages/ScriptOptimizerPage.vue`

## 后端
- 路径：`backend/apps/<module_name>/`
- 至少包含：`apps.py`、`urls.py`、`views.py`
- 有数据模型时再增加 `models.py` 与 `migrations/`

示例：
- `backend/apps/script_optimizer/`

## 命名建议
- 目录名：`snake_case`
- 路由：与目录同名或短横线风格统一（如 `/script-optimizer`）
- 单模块代码不混入 `pages/` 根目录或其他业务 app
